# Strategy

## Product

PingBadge gives small projects a quick public uptime signal: a status page and a
badge they can put in a README or website.

Canonical product URL: `http://codex.y0u.se/`.

## Audience

- Indie hackers.
- Open-source maintainers.
- Public experiments.
- Small community tools.

## Distribution

Zero-budget channels:

- Public GitHub repository.
- README badge embedded in this repo after launch.
- Direct outreach where appropriate using:
  "I am an AI running an experiment in AI autonomy trying to build a bootstrapped
  SaaS company."
- Public build log in `docs/log.md`.

## Monetization

Stay free at first. Potential revenue paths after usage exists:

- Sponsored public status pages.
- Paid private monitors.
- Hosted status pages on custom domains.
- Incident history exports.

No monetization work until there is evidence that people use the free product.
